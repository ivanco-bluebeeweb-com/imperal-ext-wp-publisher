# WP Publisher

Imperal Cloud extension that turns structured `.docx` articles into ready-to-review
WordPress drafts: Gutenberg blocks, Rank Math SEO fields, Polylang language — and
learns formatting rules from the user's confirmations (a rule goes automatic after
3 confirmations).

## How it works

1. **Deterministic parser** (`src/docx_parser.py`) — splits the document by `════`
   dividers into labelled sections (`H1:`, `LID:`, `ВЫВОД:`, `CTA:`, `SEO-БЛОК:`,
   `ИЗОБРАЖЕНИЯ:`), maps the SEO key-value block, detects subheadings heuristically
   (the source .docx carries no Word styles), and emits *warnings* for anything a
   human must answer (placeholder dates, unconfirmed subheadings).
2. **Agent (Webbee)** asks questions only for the active warnings and calls the
   typed chat functions: `parse_article`, `confirm_mapping`, `publish_draft`,
   `list_articles`.
3. **Rule base** (`src/rules.py`, `ctx.store`) — each confirmation is persisted;
   after 3 confirmations the rule switches to auto mode and the question disappears.

## Site prerequisite

The **WP Publisher Bridge** plugin (`bridge/`) must be active on the WordPress site —
it registers the `rank_math_title` / `rank_math_description` / `rank_math_focus_keyword`
meta fields in the REST API. Without it WordPress silently ignores the SEO meta.

## Configuration

Set in the extension's Secrets tab (never via chat):

- `wp_base_url` — `https://your-site.tld`
- `wp_user` — WordPress username
- `wp_app_password` — Application Password for that user

## Development

```bash
uv venv --python 3.12 venv-wp
uv pip install --python venv-wp/bin/python -r requirements-dev.txt
venv-wp/bin/pytest
venv-wp/bin/imperal validate src
venv-wp/bin/imperal build src
```

`src/imperal.json` is generated by `imperal build` — do not edit it manually.

## Layout

```
src/        extension code (main.py is the CLI/web-kernel entrypoint)
tests/      pytest suite (MockContext, no live WordPress needed)
bridge/     WP Publisher Bridge — WordPress-side helper plugin (zip + source)
docs/       project notes and the design summary
```
