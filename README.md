# WP Publisher

Imperal Cloud extension that turns structured `.docx` articles into ready-to-review
WordPress drafts: Gutenberg blocks, Rank Math SEO fields, Polylang language ‚Äî and
learns formatting rules from the user's confirmations (a rule goes automatic after
3 confirmations).

## How it works

1. **Deterministic parser** (`docx_parser.py`) ‚Äî splits the document by `‚ïê‚ïê‚ïê‚ïê`
   dividers into labelled sections (`H1:`, `LID:`, `–í–´–í–û–î:`, `CTA:`, `SEO-–ë–õ–û–ö:`,
   `–ò–ó–û–ë–Ý–ê–ñ–ï–ù–ò–Ø:`), maps the SEO key-value block, detects subheadings heuristically
   (the source .docx carries no Word styles), and emits *warnings* for anything a
   human must answer (placeholder dates, unconfirmed subheadings).
2. **Agent (Webbee)** asks questions only for the active warnings and calls the
   typed chat functions: `parse_article`, `confirm_mapping`, `publish_draft`,
   `list_articles`.
3. **Rule base** (`rules.py`, `ctx.store`) ‚Äî each confirmation is persisted;
   after 3 confirmations the rule switches to auto mode and the question disappears.

## Site prerequisite

The **WP Publisher Bridge** plugin (`bridge/`) must be active on the WordPress site ‚Äî
it registers the `rank_math_title` / `rank_math_description` / `rank_math_focus_keyword`
meta fields in the REST API. Without it WordPress silently ignores the SEO meta.

## Configuration

Set in the extension's Secrets tab (never via chat):

- `wp_base_url` ‚Äî `https://your-site.tld`
- `wp_user` ‚Äî WordPress username
- `wp_app_password` ‚Äî Application Password for that user

## Development

Create the venv **outside this directory** ‚Äî `imperal validate`/`build` walk the
whole extension root looking for Python files, and a venv nested inside it (with
its thousands of installed-package files) makes that walk take minutes instead
of seconds:

```bash
uv venv --python 3.12 ~/.venvs/wp-publisher
uv pip install --python ~/.venvs/wp-publisher/bin/python -r requirements-dev.txt
~/.venvs/wp-publisher/bin/pytest
~/.venvs/wp-publisher/bin/imperal validate
~/.venvs/wp-publisher/bin/imperal build
~/.venvs/wp-publisher/bin/python scripts/fix_manifest.py
```

`imperal.json` is generated by `imperal build` ‚Äî do not edit it manually.

## Layout

```
main.py, app.py, ...   extension code at repo root (main.py is the CLI/web-kernel entrypoint) ‚Äî
                        Imperal Cloud clones this repo and expects main.py at the root
tests/      pytest suite (MockContext, no live WordPress needed)
bridge/     WP Publisher Bridge ‚Äî WordPress-side helper plugin (zip + source)
docs/       project notes and the design summary
```
